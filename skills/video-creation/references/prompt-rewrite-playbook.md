# Prompt 改写手册

## 目标

把“生成结果不满意”拆解成可操作的改写动作，确保每轮迭代都有可验证变化。

## 改写步骤

1. 记录事实：本轮出现了什么问题。
2. 形成推断：最可能由哪类指令导致。
3. 只改一组变量：主体/动作/镜头/风格/质量约束。
4. 输出 A/B 两版改写，便于比较。

## 失败现象 -> 改写动作

### 主体不稳定

- 保留：主体身份、服饰、核心特征。
- 替换：把模糊描述改为可识别细节（颜色、材质、轮廓）。
- 加约束：`保持主角外观一致，不要切换人物身份`。

### 动作不连贯

- 保留：动作目标。
- 替换：将“大动作”拆成 2-3 段小动作，补充先后顺序。
- 加约束：`动作衔接平滑，不要突然跳帧或姿态突变`。

### 风格跑偏

- 保留：叙事内容。
- 替换：风格词从抽象词改为可参照流派或材质。
- 加约束：`保持整体风格统一，不要混入卡通化元素`。

### 细节脏乱

- 保留：构图与主体。
- 替换：减少冗余装饰词，突出主次关系。
- 加约束：`背景简洁，避免噪点和无关物体堆积`。

### 节奏失衡

- 保留：主线内容。
- 替换：明确开场 20%、中段 60%、收束 20% 的节奏。
- 加约束：`镜头节奏稳定，避免频繁突兀切换`。

## 输出模板

### 首版

```text
[主 Prompt]
{主体}，{动作}，{场景}，{镜头运动}，{风格与光线}，{质量约束}

[负面 Prompt]
{不需要元素1}，{不需要元素2}，{不需要现象3}
```

### 改写版（保留 + 替换）

```text
[保留]
- 主体：...
- 场景：...

[替换]
- 原描述：...
- 新描述：...
- 目的：...
```
