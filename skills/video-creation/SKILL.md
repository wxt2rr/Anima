# video-creation

将模糊想法转成可执行的视频生成方案，并在多轮迭代中稳定提升结果质量。

## 工作边界

- 只处理三类能力：创意、生成流程、prompt 改写。
- 默认覆盖三类输入模式：文生视频、参考图生视频、图生视频。
- 不负责剪辑、配音、字幕、发布等后期链路，除非用户明确追加。

## 输入澄清（先做）

在生成任何 prompt 前，先确认以下最小信息；缺失时先补问，不直接猜。

1. 目标：这条视频要达成什么（传播、转化、演示、叙事）。
2. 受众：面向谁看（年龄层/圈层/语言）。
3. 输出规格：时长、画幅、分辨率、节奏（慢/中/快）。
4. 风格约束：视觉风格、情绪、品牌限制、禁用元素。
5. 工具模式：文生视频 / 参考图生视频 / 图生视频。
6. 素材条件：是否有参考图、主体图、必须保留元素。

若用户信息不足，最少补问 2-3 个关键问题后再继续。

## 标准流程

1. 先产出 3 个创意方向（保守/平衡/激进），每个方向包含：
   - 一句话核心概念
   - 画面母题（主体、场景、光线、镜头语言）
   - 叙事骨架（开场-推进-收束）
2. 让用户选方向；未明确选择时，默认采用“平衡”方向。
3. 按所选工具模式输出生成方案（见下节）。
4. 产出首版 prompt 包：
   - 主 prompt
   - 负面 prompt（明确不希望出现的内容）
   - 参数建议（时长、镜头运动强度、风格强度等）
5. 根据失败现象做定向改写，而不是整体重写。

## 三种模式的生成策略

### 文生视频

- 先确定镜头与动作，再补风格细节。
- prompt 结构优先级：主体与动作 > 场景与时间 > 摄影机运动 > 风格与质感 > 质量约束。
- 适合从 1 段总 prompt + 3 段分镜 prompt 同时给出。

### 参考图生视频

- 先锁定“必须继承”的视觉特征：主体形状、配色、构图重心。
- prompt 里明确“保持一致”和“允许变化”的部分，避免模型漂移。
- 对运动描述要具体：位移方向、速度变化、镜头轨迹。

### 图生视频

- 先描述输入图的已知信息，再描述希望发生的变化。
- 避免冲突指令（例如既要求“静止构图”又要求“大幅手持晃动”）。
- 若出现形变，先降低动作复杂度，再逐步加回特效。

## Prompt 改写规则

改写目标是“针对问题收敛”，不是“词藻升级”。

1. 先定位问题类型：
   - 主体不稳定
   - 动作不连贯
   - 风格跑偏
   - 细节脏乱
   - 节奏失衡
2. 每次只改一类变量，并保留其余部分不变。
3. 使用“保留 + 替换”格式记录改动，确保可回退。
4. 连续两轮无提升时，回退到上一有效版本，改走另一创意方向。

详细改写模板见 [references/prompt-rewrite-playbook.md](references/prompt-rewrite-playbook.md)。

## 输出格式（交付给用户）

每次输出都按以下结构，避免遗漏：

1. 创意方向（3 选 1）
2. 生成流程（按所选模式）
3. 可直接使用的 prompt 包
4. 首轮参数建议
5. 失败现象到改写动作的对照表

流程检查清单见 [references/workflow-checklist.md](references/workflow-checklist.md)。

## 质量门槛

在宣称“可投喂”之前，至少自检：

- prompt 中是否同时包含主体、动作、场景、镜头、风格五要素。
- 是否明确了至少 3 个负面约束。
- 参数建议是否与目标时长和节奏一致。
- 改写建议是否能映射到具体失败现象。
